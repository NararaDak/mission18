# Mission18 프로젝트 개요 및 배경

## 1. 문서 역할

이 문서는 Mission18 프로젝트의 목적, 배경 및 전반적인 요구사항을 설명하는 문서입니다.

포함 범위:
- 프로젝트 목적 및 배경
- 프로젝트 요구사항 및 가이드라인
- 프론트엔드 및 백엔드 주요 기능

포함하지 않는 범위:
- 코딩 작성 규칙 (CONVENTION.md에서 관리)
- 기술적인 세부 사항 (TECH.md에서 관리)

## 2. 프로젝트 개요 및 목적

이 미션에서는 영화 정보, 사용자 리뷰, 리뷰 감성 분석을 표시하는 웹 애플리케이션을 개발해봅니다.
프론트엔드는 Streamlit, 백엔드는 FastAPI로 구축합니다.

*(참고: 이 레포지토리는 mission18 backend 프로젝트(`D:\project\mission18\m18_bend`)입니다. frontend 프로젝트는 `D:\project\mission18\m18_fend`에 위치합니다.)*

가이드라인:
- **(심화)**라고 작성된 부분은 선택사항입니다. 시간적으로 여유가 되면 진행해봅니다.

## 3. 기능 요구사항

### 3.1 백엔드 (FastAPI)

모든 데이터는 백엔드에서 관리합니다.

**기능:**
* **영화 관리**
  * 등록: 제목, 개봉일, 감독, 장르, 포스터 URL (나무위키 등 참고)
  * 전체 영화 / 특정 영화 조회
  * 특정 영화 삭제
* **리뷰 관리 (심화)**
  * 리뷰 등록
  * 전체 / 특정 영화 리뷰 조회
  * 리뷰 삭제
* **평점 조회**
  * 리뷰 감성 분석 점수의 평균을 통한 평점 도출
* **리뷰 감성 분석**
  * 모델: 적절한 모델을 리서치하여 적용
  * 모델 경량화 방식에 대해 고민해보기

### 3.2 프론트엔드 (Streamlit)

Streamlit 내부에서 별도의 저장 기능을 사용하지 않고, 모든 데이터 관리는 백엔드 API를 통해 이루어집니다.
배포: Streamlit Cloud

**기능:**
* **영화 목록 표시**
  * 제목, 포스터 이미지, (옵션) 평균 평점 표시
* **영화 추가**
  * 입력: 제목, 개봉일, 감독, 장르, 포스터 URL
* **리뷰 등록 (심화)**
  * 저장된 영화 선택 후 작성자 이름, 리뷰 내용 입력
* **리뷰 감성 분석**
  * 리뷰 작성 후 자동 실행되어 감성 분석 결과 표시
* **리뷰 표시**
  * 최근 10개 리뷰 표시
  * 항목: 영화 ID, 등록일, 리뷰 내용, 감성 분석 결과
